> [测试代码与你的软件使用方式越相似，
> 它们能给你的信心就越大。][guiding-principle]

该项目提供的方法和库鼓励你使用 类似用户使用页面的方式 来编写测试。

项目包含的库都是遵循以下指导原则:

1.  如果与渲染组件有关，它应该关注如何处理 DOM 节点而不是 组件实例。同时也应该不推荐处理组件实例。
2.  遵循用户使用组件的方式来编写应用组件的测试是有益的. 由于我们使用的电脑和模拟的浏览器环境，我们需要权衡，但是通常我们应该
    使用组件预期使用方式来进行测试
3.  库 和 API 应该简洁且灵活。

归根结底，我们希望这个库非常轻量、简单且易于理解。
